# Installation

GPyEDS requires Python 3.12. We strongly recommend using [uv](https://github.com/astral-sh/uv) for the fastest installation and environment management.

### Using uv (Recommended)

To install GPyEDS and create a virtual environment simultaneously:
```bash
uv pip install git+https://github.com/norberttoth398/GPyEDS.git
```

If you are developing or contributing to GPyEDS, you can clone the repository and sync the environment:
```bash
git clone https://github.com/norberttoth398/GPyEDS.git
cd GPyEDS
uv sync --all-extras
```

### Using pip

If you are not using `uv`, you can install GPyEDS using standard `pip` (ensure you are inside a Python 3.12 virtual environment):
```bash
pip install git+https://github.com/norberttoth398/GPyEDS.git
```

## Dependencies & Version Constraints

As the project evolves, you may wonder why certain dependencies or Python versions are capped. Below is a breakdown of the current dependency landscape and upstream blockers that prevent immediate upgrades.

### Python 3.12 Cap
We are currently limited to Python **3.12** instead of moving forward to Python 3.13 or 3.14. This is directly caused by a downstream limitation in `gpflow`:

1. `gpflow` inherently requires `tensorflow-macos` on Apple Silicon.
2. Google stopped publishing `tensorflow-macos` after version 2.16, as Apple Silicon support was merged directly into the mainline `tensorflow` package in version 2.17.
3. Because `tensorflow-macos` version 2.16 maxes out at Python 3.12, universal cross-platform lockfiles (like those generated by `uv`) cannot resolve the package for Python 3.13+.

*Path Forward*: Once `gpflow` updates their `pyproject.toml` to drop the deprecated `tensorflow-macos` requirement and rely entirely on `tensorflow>=2.17`, we will instantly be able to upgrade Python.

### TensorFlow Probability Cap
Because we are capped at `tensorflow` 2.16 across all platforms (due to the `tensorflow-macos` Apple Silicon limitation outlined above), we must also cap `tensorflow-probability` at `<0.25.0`.
* Versions of `tensorflow-probability` at `0.25.0` or higher enforce a strict runtime assertion requiring `tensorflow >= 2.18`. This would trigger an `ImportError` on systems running 2.16.

*Path Forward*: As soon as the `gpflow` macOS constraint is lifted, we can bump `tensorflow` to `2.18+` and instantly unlock the latest versions of `tensorflow-probability`.
